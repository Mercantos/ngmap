Basic map

Description

Shows a map with WMS layers.

Example

Code

        
          <bric-map [config]="config" [layers]="layers"></bric-map>

Properties

Property name Description Possible values
id Unique ID to identify the layer Any string value
source Define if the layer will be a BRIC pre-configured layer or a custom layer Two possible values:
  • BricMapLayerSource.bric: BRIC pre-configured layer
  • BricMapLayerSource.custom: custom layer
location Location of the service (URL) For pre-configured layers, location is not a URL, it must be one of the available servers (see list bellow):
  • urbis
  • others
For custom layers, it has to be a valid URL.
layerName The name of the layer

For pre-configured layers, it has to be one of the list:

  • municipality: Displays the municipalities limits (location must be urbis)
  • ortho: Displays the most recent aerial photographs (location must be urbis)
  • ortho2015: Displays the 2015 aerial photographs (location must be urbis)
  • ortho2012: Displays the 2012 aerial photographs (location must be urbis)
  • regional-roads: Displays the regional roads layer (location must be urbis)

If a string is passed, the string is used as the LAYERS parameter in the request to the server
alpha The layer opacity Any value between 0 and 1. 0 is transparent, 1 is full opacity. Default is 1.
visible The layer visibility A boolean value. True makes the layer visible, False makes it invisible. Default is true.
projection The projection to use with the layer Any valid projection. Default is EPSG:31370
type The type of layer For custom layers only. Defines if the layer is a WMS (pictures) or a WFS (data). Possible values are:
  • BricMapLayerType.wms
  • BricMapLayerType.wfs
params List of parameters for the layer For WMS custom layers only. A key-value object that will be sent as layer parameters. Can't contain the 'LAYERS' key.

Events

The bric-map component fires an event mapInitialized when the map is initializated. It returns the ol.map object.

The object can be used to manipulate the map and layers in order to implement things that the component doesn't include.

Usage : 

    
      <bric-map [config]="config" [layers]="layers" (mapInitialized)="myMapObject"></bric-map>

Notes

The layers are displayed in the same order as the layers attribute. The index 0 is the bottom layer, the biggest index is on top.

It's possible to add or remove layers from the layers property but changing order will result in unpredictable result.

The properties source, projection and type can't be changed during execution. Other properties can be changed and will immediately affect the map.
